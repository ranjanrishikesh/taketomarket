Ask user: "What would you like to audit? Provide a URL, content file path, or paste content directly."

Parse input:
- If URL: attempt WebFetch (MCP detection). If unavailable, ask user to paste the page HTML/content.

---
## Step 3: Run SEO Gate Checks

Evaluate content against each SEO discipline gate:

1. **Search Intent Match** - Does the content satisfy the likely search intent (informational, transactional, navigational)?
2. **Keyword Placement** - Primary keyword in: title tag, H1, first 100 words, meta description, URL slug
3. **Content Structure** - Proper heading hierarchy (H1 > H2 > H3), no skipped levels, scannable structure
4. **Internal Linking Density** - At least 3 internal links per 1000 words (or links present if content is shorter)
5. **Schema Markup** - Appropriate schema.org markup present (Article, FAQ, HowTo, Product as relevant)
6. **Entity Coverage** - Key entities mentioned that search engines associate with the topic
7. **Thin Content Detection** - Word count adequate for topic depth (>300 for blog, >800 for pillar, >150 for landing)
8. **Core Web Vitals Budget** - Image count/size reasonable, no render-blocking patterns, lazy loading present

Per check: assign PASS / WARN / FAIL with specific evidence quote or observation.
---
## Step 4: Generate Report

Display structured report:
```
========================================
takeToMarket > SEO AUDIT REPORT
========================================

Target: [URL or filename]
Overall: [X/8 PASS] [Y/8 WARN] [Z/8 FAIL]

| # | Gate | Result | Evidence |
|---|------|--------|----------|
| 1 | Search Intent Match | PASS/WARN/FAIL | [brief evidence] |
...

FINDINGS (WARN/FAIL details):
- [Gate]: [Specific issue and recommendation]

RECOMMENDATIONS:
1. [Priority fix #1]
2. [Priority fix #2]
...

## Step 3: Run Email Preflight Checks

Evaluate against email discipline gates:
