  - `sourceAssetId` (number): the asset_id of the source asset being repurposed

## Step 1: Load Context

## Step 2: Identify Source Asset

Parse $ARGUMENTS for [source-asset] path.

Validate:
1. File exists at the specified path
2. File is within a `.marketing/CAMPAIGNS/[slug]/ASSETS/` directory
3. Campaign is in appropriate phase (produced, verified, reviewed, fixed, or shipped)

Extract:
- CAMPAIGN_SLUG from path
- SOURCE_ASSET_ID from MANIFEST.json (match by filename)
- SOURCE_CHANNEL from MANIFEST.json asset entry
- SOURCE_CONTENT by reading the file

If validation fails: error with specific message about what's wrong and how to fix.

---

## Step 3: Select Target Channels

Read .marketing/CHANNELS.md active channels list.
Remove SOURCE_CHANNEL from the list (don't repurpose to same channel).

Default: all remaining active channels.

Ask user (per D-12):
"Repurpose to which channels? Default: all active channels minus source."
Present channel list with numbers for selection.
Options: "all" (default), or comma-separated numbers to pick specific channels.

Identify HERO_CHANNEL: highest-reach channel from CHANNELS.md baselines (per D-13).

---

## Step 4: Generate Derivative Briefs

For each target channel, generate a mini-brief:

```
# Derivative Brief: [channel]
Source: [source asset filename]
Campaign: [CAMPAIGN_SLUG]

## Adaptation Requirements
- Channel: [channel name]
- Format: [format from channel config in CHANNELS.md]
- Length: [typical length for this channel]
- Tone adjustment: [per BRAND.md tone-per-context if defined for this channel]

## Core Message (preserve from source)
[Extract the core message/thesis from source asset]

## Channel-Specific Adaptations
[Based on relevant playbook: format rules, hook requirements, length constraints]

## Positioning Anchor
[From POSITIONING.md -- must be maintained in derivative]

## Outcome Metric
[Inherit from campaign brief OR assign channel-default metric]
```

Write each brief to: `.marketing/CAMPAIGNS/${SLUG}/REPURPOSE-BRIEF-[channel-slug].md`

---

## Step 5: Produce Hero Derivative

```
takeToMarket > REPURPOSING: HERO DERIVATIVE ([HERO_CHANNEL])
```

Read the agent prompt template from `${CLAUDE_PLUGIN_ROOT}/agents/ttm-producer.md`.

Fill placeholders:

- `[POSITIONING_PATH]` --> `.marketing/POSITIONING.md`
- `[BRAND_PATH]` --> `.marketing/BRAND.md`
- `[ICP_PATH]` --> `.marketing/ICP.md`
- `[PLAYBOOK_PATH]` --> resolved playbook for HERO_CHANNEL
- `[OUTPUT_PATH]` --> `.marketing/CAMPAIGNS/${SLUG}/ASSETS/R-01-${HERO_CHANNEL_SLUG}.md`

Also inject: "SOURCE ASSET for reference: [SOURCE_CONTENT or path]"

Call Task() with the populated prompt.
WAIT for Task() to complete before proceeding.

---

## Step 6: Produce Remaining Derivatives (Wave-Parallel)

If only 1 target channel (hero only): skip this step.

```
takeToMarket > REPURPOSING: DERIVATIVES ([N] channels, wave-parallel)
```

For each remaining target channel:
1. Read agent prompt template
2. Fill placeholders with channel-specific values
3. Set output path: `.marketing/CAMPAIGNS/${SLUG}/ASSETS/R-[NN]-${CHANNEL_SLUG}.md`
4. Call Task() with populated prompt

All derivative Task() calls run in parallel (per D-13).
After ALL complete, verify each output file exists and has content (> 50 chars).

Run:

node "${CLAUDE_PLUGIN_ROOT}/bin/ttm-tools.cjs" campaign repurpose-manifest "${SLUG}" ${SOURCE_ASSET_ID} '${DERIVATIVES_JSON}'
