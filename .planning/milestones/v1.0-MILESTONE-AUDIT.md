## Integration Check Results

### Blockers (3)

#### B-01: learn→archive lifecycle break
- **From:** Phase 9 (learn.md line 353)
- **To:** Phase 7 (archive.md, campaign.cjs)
- **Issue:** learn.md sets `phase.learned true` but never `campaign update phase learned`. Archive only accepts `shipped`. Full lifecycle (ship→measure→learn→archive) breaks at learn→archive.
- **Affected:** LRNG-01, LRNG-02, LRNG-03, LRNG-04, STAT-03
- **Fix:** (1) Add `campaign update ${SLUG} phase learned` to learn.md (2) Update archive to accept `learned` in addition to `shipped`

#### B-02: install.js missing agents/ directory
- **From:** Phase 10 (install.js line 13-22)
- **To:** Phase 4 (produce.md, fix.md, repurpose.md)
- **Issue:** DIRS_TO_COPY omits `agents/`. npm/git-clone installs miss `agents/ttm-producer.md`. Every `/ttm-produce`, `/ttm-fix`, `/ttm-repurpose` invocation fails post-install.
- **Affected:** DIST-01, DIST-02, DIST-03, LIFE-06
- **Fix:** Add `'agents'` to DIRS_TO_COPY array in install.js

#### B-03: Verification filename mismatch
- **From:** Phase 4 (verify.md writes `VERIFICATION.md`)
- **To:** Phase 9 (learn.md references `VERIFY-REPORT-*.md`), Phase 7 (archive.md same)
- **Issue:** learn.md and archive.md glob for `VERIFY-REPORT-*.md` which never exists. Gate failure details silently missing from lesson extraction.
- **Affected:** LRNG-02, STAT-05
- **Fix:** Change `VERIFY-REPORT-*.md` to `VERIFICATION.md` in learn.md, archive.md, learnings-extraction.md
