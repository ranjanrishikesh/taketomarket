## Step 3: Detect Analytics Source (per D-01, D-04)
Attempt detection in priority order:

### Priority 1: MCP Analytics Tools (ANALYTICS_MODE=mcp)
Check for analytics MCP tools by attempting tool availability detection:
- PostHog MCP tools (posthog_*, *posthog*)
- Amplitude MCP tools
- GA4 / Google Analytics MCP tools
- Any tool matching pattern *analytics*, *metric*, *event*

If any analytics MCP tool detected: set ANALYTICS_MODE=mcp, proceed to Step 4 with MCP collection.

### Priority 2: CSV/Markdown Paste (ANALYTICS_MODE=paste)
If no MCP tools detected, prompt user:
```
takeToMarket > NO ANALYTICS MCP TOOLS DETECTED

You can paste your analytics data directly. We accept:
- CSV data (comma or tab separated)
- Markdown tables
- Plain text with metric labels and values

Here's what we need (paste all available data):
```
Display the contents of references/measurement-template.md.
Set ANALYTICS_MODE=paste.

### Priority 3: Structured Batch Questions (ANALYTICS_MODE=batch)
If user indicates they cannot paste bulk data, ask in BATCHES per D-02 (not one question at a time):

**Batch 1 -- Outcome Metrics:**
"What are your results for the campaign's outcome metric(s)? [list outcome metrics from brief with their targets]"

**Batch 2 -- Traffic & Engagement:**
"Paste your traffic metrics (page views, sessions, unique visitors, bounce rate, avg time on page)"

**Batch 3 -- Conversion & Revenue:**
"Paste your conversion metrics (conversions, conversion rate, revenue, pipeline value)"

**Batch 4 -- Channel Attribution:**
"For each channel used in this campaign, share: touches/impressions, conversions attributed, revenue attributed"

## Step 4: Collect and Parse Data
Based on ANALYTICS_MODE:

- **mcp:** Use detected MCP tools to query analytics data for the campaign's measurement window. Map tool output to expected metric fields.
- **paste:** Parse pasted CSV/Markdown tables. Map columns to expected metric fields. If unmapped columns exist per D-03, ask user: "I found columns [X, Y, Z] that I couldn't map. What do these represent?"
- **batch:** Collect responses from all 4 batches. Structure into the same internal format.

## Step 5: Apply Attribution Models
Calculate all 3 models from channel attribution data:

### Last-Touch Attribution
Assign 100% credit to the last touchpoint before each conversion.

### Linear Attribution
Distribute credit equally across all touchpoints in each conversion path.

### Time-Decay Attribution (DEFAULT DISPLAY per D-05)
Weight credit exponentially toward the most recent touchpoint. Half-life: 7 days.

If channel attribution data is insufficient (single channel only), note: "Single-channel campaign -- attribution models show identical results."

## Step 6: Generate Measurement Report (Outcome-First per D-05, D-06, LIFE-15)
Display: `takeToMarket > GENERATING MEASUREMENT REPORT`

Generate MEASUREMENT.md using templates/measurement-report.md structure. **CRITICAL: Outcome Assessment section MUST be first.** Open with "Did we hit the target?" followed by YES/NO.

Write to: `.marketing/CAMPAIGNS/${SLUG}/MEASUREMENT.md`
