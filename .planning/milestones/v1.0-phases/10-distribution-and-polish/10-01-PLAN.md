  <action>
Create `install.js` at project root. This is a Node.js CJS script with shebang `#!/usr/bin/env node`.

Structure (per D-01 through D-05):

1. **Constants:**
   - `PACKAGE_ROOT = __dirname` (works for both local and npx execution)
   - `DIRS_TO_COPY = ['.claude-plugin', 'skills', 'workflows', 'templates', 'references', 'playbooks', 'gates', 'bin']`
   - `FILES_TO_COPY = ['settings.json']`

2. **detectRuntime(args) function** (per D-02):
   - Parse `--runtime claude|codex` from args (index of `--runtime` + next arg)
   - If no flag: check `path.join(os.homedir(), '.claude')` exists -> return 'claude'
   - Then check `path.join(os.homedir(), '.codex')` exists -> return 'codex'
   - Default: return 'claude' with `console.log('Note: Defaulting to Claude Code. Use --runtime codex if using Codex.')`

3. **copyDirSync(src, dest) function:**
   - `fs.mkdirSync(dest, { recursive: true })`
   - Iterate `fs.readdirSync(src, { withFileTypes: true })`
   - For directories: recurse. For files: `fs.copyFileSync`
   - Security: Skip symlinks (`entry.isSymbolicLink()`) with warning per RESEARCH threat model

4. **validateInstall(targetDir) function** (per D-03):
   - Check each DIRS_TO_COPY exists in targetDir
   - Check `plugin.json` exists at `path.join(targetDir, '.claude-plugin', 'plugin.json')`
   - Check at least 5 SKILL.md files exist under `path.join(targetDir, 'skills')`
   - Return array of `{ name, status: 'pass'|'fail' }` objects

5. **main() function** (per D-04):
   - Parse args from `process.argv.slice(2)`
   - Check for `--dry-run` flag in args. If present, set `DRY_RUN = true`
   - Call `detectRuntime(args)`
   - Compute targetDir: `path.join(os.homedir(), runtime === 'codex' ? '.codex' : '.claude', 'plugins', 'taketomarket')`
   - Print banner: `\ntakeToMarket installer\nRuntime: ${runtime}\nTarget: ${targetDir}\n`
   - Check if targetDir already exists. If yes, print "Existing installation found. Overwriting..."
   - If DRY_RUN: skip actual file copy, run validateInstall against PACKAGE_ROOT instead (validates source completeness without writing to target), print "[DRY RUN] No files written.", and exit(0) after printing validation results
   - If NOT DRY_RUN: Copy all DIRS_TO_COPY from PACKAGE_ROOT to targetDir (skip if source dir doesn't exist)
   - Copy all FILES_TO_COPY
   - Call validateInstall(targetDir)
   - Print results table: each check as `[PASS] dirname` or `[FAIL] dirname`
   - Count failures. If any: print "Installation incomplete. Some components missing." and exit(1)
   - If all pass: print quickstart block:
     ```
     Installation complete!

     Quick start:
       1. Open a project directory
       2. Run /ttm-init to set up your marketing workspace
       3. Run /ttm-new-campaign <name> to start your first campaign

     Documentation: https://github.com/[repo]/blob/main/README.md
     ```

6. Call `main()` at bottom.

Use `'use strict'` at top. Only require `fs`, `path`, `os` from Node built-ins.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User CLI args -> install.js | Untrusted --runtime value and any other args from user |
| install.js -> filesystem | File copy operations write to user's home directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | install.js path resolution | mitigate | Use path.resolve() + validate targetDir starts with os.homedir(); never allow ../ traversal in runtime arg |
| T-10-02 | Tampering | copyDirSync symlink following | mitigate | Check entry.isSymbolicLink() before copy; skip symlinks with console.warn |
| T-10-03 | Tampering | Arbitrary file overwrite | mitigate | Only write to known plugin subdirectory (plugins/taketomarket/); never overwrite files outside this path |
| T-10-04 | Denial of Service | Large file copy | accept | Plugin files are small Markdown; no risk of disk exhaustion |
</threat_model>
