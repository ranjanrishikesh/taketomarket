<purpose>
On-demand competitor analysis that updates COMPETITORS.md with new intelligence.

paste when tools unavailable. Validates findings against POSITIONING.md to identify
differentiation gaps and opportunities.
</purpose>

<required_reading>

</required_reading>

<constraints>
## POSITIONING.md is READ-ONLY
[standard read-only constraint block]
</constraints>

<process>
## Text-Mode Detection
[standard text-mode block]
---
## Step 1: Load Context

Load Tier 1 summaries from all 9 reference files.
Load Tier 2 (full content): `.marketing/COMPETITORS.md` and `.marketing/POSITIONING.md`

---
## Step 2: Tool Detection

Attempt a WebSearch call with a minimal test query related to the user's category (from POSITIONING.md category field).

### When WebSearch is available (SEARCH_MODE=web)

```
takeToMarket > COMPETITOR SCAN MODE: WEB
```

### When WebSearch is NOT available (SEARCH_MODE=manual)

```
takeToMarket > COMPETITOR SCAN MODE: MANUAL
```

Tell the user:
```
Web search tools are not available in this session.
To run a competitor scan, please paste any of the following:

1. Competitor websites or landing page content
2. Competitor social media posts or ad creative
3. Industry reports or market analysis
4. Search results for your category keywords
5. Any other competitive intelligence you've gathered

Paste your findings below and I will analyze them.
```

---
## Step 3: Research Competitors

### If SEARCH_MODE=web:
- Search for each competitor listed in COMPETITORS.md (name + category keywords)
- Search for new entrants: "[category] alternatives [current year]"
- Use WebFetch on competitor homepages for positioning analysis
- Tag findings with confidence: HIGH (verified URL), MEDIUM (indirect), LOW (inference)

### If SEARCH_MODE=manual:
- Analyze pasted content for: positioning claims, messaging themes, channel presence, content strategy, pricing signals

---
## Step 4: Analyze and Compare

For each competitor (existing and newly discovered):
- Identify their current positioning claim
- Compare against user's POSITIONING.md differentiator
- Flag overlap areas (where competitor messaging is similar to user's)
- Identify gaps (areas competitors aren't covering that user could own)
- Note share-of-voice signals if available

---
## Step 5: Validate and Update

Ask user to confirm:
- New competitors to add
- Updated positioning maps for existing competitors
- Any competitors to mark as inactive/irrelevant

Validate: no proposed update contradicts POSITIONING.md (e.g., suggesting user adopt a competitor's positioning).

Write updated `.marketing/COMPETITORS.md`:
- Update existing competitor entries with new findings
- Add new competitors with full profiles
- Mark inactive competitors with `[INACTIVE: YYYY-MM-DD]`
- Update "Positioning Map" section
- Preserve file structure, frontmatter, and summary markers

---
## Step 6: Completion Banner

```
========================================
takeToMarket > COMPETITOR SCAN COMPLETE
========================================

Competitors analyzed: ${COUNT}
New competitors found: ${NEW_COUNT}
Positioning overlaps detected: ${OVERLAP_COUNT}

Next steps:
- Review updated .marketing/COMPETITORS.md
- Run /ttm-positioning-check if overlaps concern you
```
</process>
