## Command Reference

### Campaign Lifecycle
| Command | Description |
|---------|-------------|
| `/ttm-init` | One-time setup: generates all reference files from guided interview |
| `/ttm-new-campaign <slug>` | Creates campaign directory with initialized state |
| `/ttm-research <slug>` | Market/audience research with web search or manual paste |
| `/ttm-brief <slug>` | Generates brief with outcome metric enforcement |
| `/ttm-produce <slug>` | Content production in fresh isolated contexts |
| `/ttm-verify <slug>` | 10-gate quality wall evaluation |
| `/ttm-review <slug>` | Human review with structured checklist |
| `/ttm-fix <slug>` | Root-cause fix loop (3-attempt cap) |
| `/ttm-ship <slug>` | Launch checklist and shipping |
| `/ttm-measure <slug>` | Analytics analysis with attribution models |
| `/ttm-learn <slug>` | Lesson extraction and reference file updates |

### State Management
| Command | Description |

| `/ttm-health` | Validate .marketing/ directory integrity |
| `/ttm-next` | Guidance on the right next command |

## Quality Gate Wall

Every asset passes through 10 base quality gates:

1. **Positioning Drift** (Tier 1 - blocking)
2. **Claim Accuracy** (Tier 1 - blocking)
3. **Voice Drift** (Tier 2 - advisory)
4. **Outcome Alignment** (Tier 1 - blocking)
5. **Funnel Integrity** (Tier 2 - advisory)
6. **UTM Hygiene** (Tier 2 - advisory)
7. **Compliance** (Tier 2 - advisory)
8. **Competitor Collision** (Tier 2 - advisory)
9. **ICP Fit** (Tier 2 - advisory)
10. **Format Correctness** (Tier 2 - advisory)

Discipline playbooks add channel-specific gates on top.

## Playbooks

10 discipline playbooks with channel-specific gates:
- SEO, AEO, Email, LinkedIn, Social, YouTube, Paid Ads, Affiliate, PR/Media, Events

## Architecture (for contributors)

```
.claude-plugin/         Plugin manifest
skills/                 SKILL.md command stubs (routing only)
workflows/
  setup/               /ttm-init interview workflow
  lifecycle/           Campaign phase workflows (research, brief, produce, verify, review, fix, ship, measure, learn)
  reference-mgmt/      Reference file management (positioning-check, positioning-shift, brand-refresh, icp-refresh, competitor-scan)
  discipline/          Discipline utility workflows (seo-audit, aeo-check, keyword-map, email-preflight, affiliate-kit, repurpose)
  utility/             State management workflows (state, resume, archive, health, next)
templates/             Markdown templates for generated files
references/            Domain knowledge and context loading rules
playbooks/             Discipline playbook files (seo, aeo, email, linkedin, social, youtube, paid-ads, affiliate, pr-media, events)
gates/                 Gate evaluation logic
agents/                Subagent prompt templates (producer, verifier)
bin/
  ttm-tools.cjs       CLI entry point
  lib/                 CJS modules (campaign, health, core, slug, state, drift-log, deviation, commit)
```

### Key Patterns
- **Thin SKILL.md routing:** Each skill is <30 lines, routes to a workflow file
- **500-line file limit:** Large content extracted to @-referenced files
- **Two-tier context loading:** Compact summaries load universally, full docs load when needed
- **Positioning-as-invariant:** POSITIONING.md is read-only during campaigns
- **Zero dependencies:** Only Node.js built-ins in bin/ tools
