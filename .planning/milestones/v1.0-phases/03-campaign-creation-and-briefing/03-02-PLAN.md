**Step 5: Update global state**
- Run `node ${CLAUDE_PLUGIN_ROOT}/bin/ttm-tools.cjs state update current_campaign ${CAMPAIGN_SLUG}`

**Step 6: Display summary**
- Show the created structure:
  ```
  takeToMarket > CAMPAIGN CREATED: ${CAMPAIGN_SLUG}
  ```

Create `workflows/lifecycle/research.md` following the init.md structural pattern. This is the /ttm-research workflow implementing D-03 (web search + manual paste hybrid) and D-04 (structured output with confidence scores).

The workflow has 7 steps:

**Step 1: Load context**
- Read Tier 1 summaries (content between `<!-- _SUMMARY -->` and `<!-- END_SUMMARY -->`) from all 9 `.marketing/` reference files: POSITIONING.md, BRAND.md, ICP.md, CHANNELS.md, STATE.md (frontmatter only), CALENDAR.md, COMPETITORS.md, METRICS.md, LEARNINGS.md
- Read Tier 2 (full content) for: `.marketing/COMPETITORS.md` (per context-loading.md loading matrix -- research loads Tier 2 COMPETITORS)
- Display stage banner: `takeToMarket > LOADING CONTEXT`

**Step 2: Validate campaign**
- Extract slug from $ARGUMENTS (strip --text flag if present)
- Run `ls .marketing/CAMPAIGNS/$SLUG/STATE.md 2>/dev/null && echo "exists" || echo "missing"`
- If "missing": tell user campaign does not exist, suggest running /ttm-new-campaign first. Exit.
- Read campaign STATE.md: `node ${CLAUDE_PLUGIN_ROOT}/bin/ttm-tools.cjs campaign state $SLUG --raw`

**Step 3: Phase order check (per D-08)**
- Check the `phase` field from campaign state
- If phase is NOT "created": warn user: "Campaign is already in phase [X]. Running research again will overwrite RESEARCH.md. Proceed anyway?"
- Wait for user confirmation before continuing

**Step 4: Understand campaign**
- Read the campaign name and any goal information from campaign STATE.md
- Ask user (freeform, not AskUserQuestion): "What is the topic or focus of this campaign? What keywords or themes should I research?"
- Store the response as RESEARCH_TOPIC

**Step 5: Tool detection and research (per D-03)**
- Display stage banner based on mode
- Attempt a WebSearch call to detect tool availability:

When WebSearch is available (SEARCH_MODE=web):
  - Display `takeToMarket > RESEARCH MODE: WEB`
  - Use WebSearch for SERP queries based on RESEARCH_TOPIC and campaign keywords
  - Use WebFetch for competitor page content extraction
  - Cross-reference findings against COMPETITORS.md
  - Tag insights: HIGH confidence (verified source URL), MEDIUM (indirect evidence), LOW (inference)

When WebSearch is NOT available (SEARCH_MODE=manual):
  - Display `takeToMarket > RESEARCH MODE: MANUAL`
  - Tell user: "Web search tools are not available in this session. To get the best research, please paste:"
    1. Google search results for: [3 suggested queries based on RESEARCH_TOPIC]
    2. Any competitor blog posts or landing pages you want analyzed
    3. Reddit/forum discussions about [RESEARCH_TOPIC]
    4. Any other market data you have
  - "Paste your findings and I will analyze them."
  - Wait for user to paste content
  - Analyze pasted content against reference files
  - Tag all pasted insights as MEDIUM confidence (user-provided, not independently verified)

**Step 6: Generate RESEARCH.md**
- Display `takeToMarket > GENERATING RESEARCH`
- Read the template: `${CLAUDE_PLUGIN_ROOT}/templates/campaign-research.md`
- Fill all sections with research findings:
  - Replace `[CAMPAIGN_NAME]` with campaign name
  - Replace `[SLUG]` with campaign slug
  - Set **Researched:** to current timestamp
  - Set **Method:** to web-search, manual-paste, or hybrid
  - Market Context: fill insight table with findings and confidence tags
  - Competitor Content Analysis: fill competitor table from research + COMPETITORS.md
  - Audience Insights: fill insight table from ICP.md + research findings
  - Ambient Narrative: write 2-3 paragraphs about what the market already believes
  - Content Gaps: fill gaps table with opportunities
  - Research Summary: write 2-3 sentence summary that /ttm-brief will consume
- Write to `.marketing/CAMPAIGNS/${SLUG}/RESEARCH.md`

**Step 7: Update campaign state**
- Run `node ${CLAUDE_PLUGIN_ROOT}/bin/ttm-tools.cjs campaign update ${SLUG} phase researched`
- Run `node ${CLAUDE_PLUGIN_ROOT}/bin/ttm-tools.cjs campaign update ${SLUG} phase.researched $(node ${CLAUDE_PLUGIN_ROOT}/bin/ttm-tools.cjs timestamp --raw)`
- Display summary:
  ```
  takeToMarket > RESEARCH COMPLETE

  Research saved to .marketing/CAMPAIGNS/${SLUG}/RESEARCH.md
  Method: [web-search|manual-paste|hybrid]
  Insights: [count] with confidence breakdown

  Next: Run /ttm-brief ${SLUG}
  ```

- [ ] RESEARCH.md written to campaign directory
- [ ] Campaign state updated to phase=researched
