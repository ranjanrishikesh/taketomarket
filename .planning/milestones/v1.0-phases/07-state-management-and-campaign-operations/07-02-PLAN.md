**workflows/utility/next.md** -- Create workflow:

`<purpose>`: Next-command routing workflow for /ttm-next. Looks across ALL active campaigns, prioritizes which needs attention most per D-14, and suggests the specific /ttm-* command to run per D-15 priority algorithm. Outputs prioritized list with top recommendation per D-16.

`<required_reading>`: `@${CLAUDE_PLUGIN_ROOT}/references/context-loading.md`

`<constraints>`: POSITIONING.md read-only block.

`<process>`:

**Step 1: Load All Campaigns** -- Banner: `takeToMarket > SCANNING CAMPAIGNS`
- Run: `node "${CLAUDE_PLUGIN_ROOT}/bin/ttm-tools.cjs" campaign list --raw`
- Parse JSON. Filter out `archived` and `cancelled` campaigns. IMPORTANT: Do NOT use `--active` flag because it excludes `created` and `researched` phases (per Pitfall 2 in RESEARCH.md). Use unfiltered list and filter in the workflow.

**Step 2: Determine Next Command Per Campaign** -- Banner: `takeToMarket > ANALYZING PRIORITIES`
- For each non-archived, non-cancelled campaign, determine the next command using the phase-to-command mapping:
  - `created` -> `/ttm-research <slug>`
  - `researched` -> `/ttm-brief <slug>`
  - `briefed` -> `/ttm-produce <slug>`
  - `produced` -> `/ttm-verify <slug>`
  - `verified` -> `/ttm-review <slug>`
  - `reviewed` -> Check `review.overall_result`: if `'revise'` -> `/ttm-fix <slug>`, if `'approved'` -> `/ttm-ship <slug>`
  - `fixed` -> `/ttm-review <slug>` (re-review after fix)
  - `shipped` -> `/ttm-measure <slug>` (Phase 9, note: not yet implemented)
  - `measured` -> `/ttm-learn <slug>` (Phase 9, note: not yet implemented)
  - `learned` -> `/ttm-archive <slug>`

- For interrupted verify/fix loops per D-05: if `fix.run_count` > 0 AND `review.overall_result` = `'revise'`, suggest continuing the fix loop with `/ttm-fix <slug>` rather than restarting.

**Step 3: Apply Priority Algorithm** (per D-15):
- Priority 1: Campaigns with `review.overall_result = 'approved'` (pending human action to ship)
- Priority 2: Campaigns with fix loops in progress (`fix.run_count > 0`, `review.overall_result = 'revise'`)
- Priority 3: Campaigns at earlier lifecycle phases (use phase order index: created=0, researched=1, briefed=2, ... lower index = earlier = higher priority)
- Priority 4: Most recently active (`last_updated` most recent)
- Tie-break: Campaign creation date (oldest first, per D-15)

**Step 4: Display Recommendations** -- Banner: `takeToMarket > NEXT ACTIONS`
- If no active campaigns: display message "No active campaigns found. Run `/ttm-new-campaign <slug>` to start one."
- If campaigns exist per D-16:
```
## Recommended Next Action

**Run:** `/ttm-fix spring-launch`
**Why:** Fix loop in progress (attempt 2 of 3). Review found positioning drift in blog post.

---

### Other Active Campaigns

| Priority | Campaign | Phase | Suggested Command | Reason |
|----------|----------|-------|-------------------|--------|
| 2 | summer-promo | briefed | /ttm-produce summer-promo | Ready for content production |
| 3 | q3-launch | created | /ttm-research q3-launch | New campaign, needs research |
```

- Show 1 primary recommendation highlighted (per D-16), then up to 3 alternatives in a table.

`<success_criteria>`: Prioritized recommendation displayed with reasoning.
