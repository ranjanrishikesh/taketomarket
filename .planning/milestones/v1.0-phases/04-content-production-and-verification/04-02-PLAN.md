**Steps inside `<process>` (8 steps):**

**Text-Mode Detection** — copy the exact pattern from brief.md lines 18-43. Produce does not use AskUserQuestion, but include the detection for consistency and future use.

**Step 1: Load Context** — follow brief.md Step 1 pattern:
- Display: `takeToMarket > LOADING CONTEXT`
- Extract slug: `SLUG=$(echo "$ARGUMENTS" | sed 's/--text//g' | xargs)`
- Load Tier 1 summaries from all 9 reference files (list them explicitly per context-loading.md)
- Load Tier 2 full content for: POSITIONING.md, BRAND.md, ICP.md (per D-03)
- Read campaign files: CAMPAIGNS/${SLUG}/STATE.md, CAMPAIGNS/${SLUG}/BRIEF.md

**Step 2: Validate Campaign State** — follow brief.md campaign validation pattern:
- Run `node "${CLAUDE_PLUGIN_ROOT}/bin/ttm-tools.cjs" campaign state "${SLUG}" --raw`
- Check `exists: false` -> error with "Campaign not found" and suggest /ttm-new-campaign
- Check `phase` field -> warn if not `"briefed"` (expected predecessor for produce)
- If phase is already "produced" or later, warn user and ask for confirmation before re-producing

**Step 3: Parse Assets List from Brief** — NEW (no direct brief.md analog):
- Display: `takeToMarket > PARSING BRIEF`
- Read CAMPAIGNS/${SLUG}/BRIEF.md
- Extract the assets list from the brief (the "Assets List" section created by /ttm-brief)
- Each asset entry should have: type (blog-post, linkedin-post, email, etc.), channel, and whether it is the hero asset
- The FIRST asset in the list is the hero asset (per D-01)
- If no assets list found in brief, error with "Brief has no assets list — run /ttm-brief first"

**Step 4: Resolve Playbooks** — NEW:
- For each asset, map its type to a playbook file path: `${CLAUDE_PLUGIN_ROOT}/playbooks/<type>.md`
- Check if playbook file exists using Bash `test -f`
- If playbook exists: record path for context loading
- If playbook does NOT exist: log warning `"No playbook found for [type] — producing with base context only"` and set playbook to "none" (per RESEARCH.md Pitfall 5 graceful fallback)
- This is the loading MECHANISM only — playbook content is Phase 8

**Step 5: Produce Hero Asset** — Core production step:
- Display: `takeToMarket > PRODUCING HERO ASSET`
- Read the agent prompt template from `${CLAUDE_PLUGIN_ROOT}/agents/ttm-producer.md`
- Fill all placeholders with actual paths:
  - [BRIEF_PATH] -> `.marketing/CAMPAIGNS/${SLUG}/BRIEF.md`
  - [POSITIONING_PATH] -> `.marketing/POSITIONING.md`
  - [BRAND_PATH] -> `.marketing/BRAND.md`
  - [ICP_PATH] -> `.marketing/ICP.md`
  - [PLAYBOOK_PATH] -> resolved playbook path or "none"
  - [OUTPUT_PATH] -> `.marketing/CAMPAIGNS/${SLUG}/ASSETS/01-<type>-<channel>.md`
  - [ASSET_TYPE] -> hero asset type from brief
  - [CHANNEL] -> hero asset channel from brief
- Call Task() with the populated prompt
- WAIT for Task() to complete (blocking — hero must finish before derivatives)
- After Task() returns, VERIFY the hero file exists and has content:
  ```bash
  test -s ".marketing/CAMPAIGNS/${SLUG}/ASSETS/01-<hero-filename>.md"
  ```
  If file is empty or missing, report error: "Hero asset production failed — file not written"
- This verification step mitigates RESEARCH.md Pitfall 2 (race condition on hero write)

**Step 6: Produce Derivative Assets** — Wave-parallel production:
- Display: `takeToMarket > PRODUCING DERIVATIVES (${N} assets)`
- For each derivative asset in the brief's assets list (all except hero):
  - Read agent prompt template
  - Fill placeholders (same as Step 5 but with derivative-specific type/channel/output path)
  - Add the hero asset file path to the prompt: "Also read [HERO_PATH] as the anchor asset for message consistency"
  - Assign sequential filenames: `02-<type>-<channel>.md`, `03-<type>-<channel>.md`, etc.
  - Call Task() for each derivative
- All derivative Task() calls can run in parallel (per D-01 wave-parallel)
- After all Task() calls complete, verify each derivative file exists and has content
- If any derivative failed, report which ones failed but continue with the rest

**Step 7: Write Production Manifest** — Bridge to verify:
- Display: `takeToMarket > WRITING MANIFEST`
- Read template from `${CLAUDE_PLUGIN_ROOT}/templates/production-manifest.json`
- Fill with actual values: slug, timestamp, context paths, hero details, derivative details, total_assets count
- Write to `.marketing/CAMPAIGNS/${SLUG}/MANIFEST.json`
- The manifest is the ONLY bridge between produce and verify contexts (per D-11)

**Step 8: Update Campaign State and Summary** — follow brief.md state update pattern:
- Update campaign phase: `node "${CLAUDE_PLUGIN_ROOT}/bin/ttm-tools.cjs" campaign update "${SLUG}" phase produced`
- Update phase timestamp: `node "${CLAUDE_PLUGIN_ROOT}/bin/ttm-tools.cjs" campaign update "${SLUG}" phase.produced "$TIMESTAMP"`
- Display completion banner:
  ```
  takeToMarket > PRODUCTION COMPLETE

  Assets produced: [N]
  Hero: [hero filename]
  Derivatives: [list of derivative filenames]
  Manifest: .marketing/CAMPAIGNS/${SLUG}/MANIFEST.json

  Next: Run /ttm-verify ${SLUG} to validate assets against quality gates
  ```
