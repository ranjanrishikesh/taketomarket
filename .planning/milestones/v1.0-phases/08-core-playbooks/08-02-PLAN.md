**## Production Guidance** -- How to write SEO content. Cover:
- Search intent first: identify whether the query is informational, commercial, navigational, or transactional. Match the content format to the intent.
- Keyword placement: target keyword in title, H1, first 100 words, and at least one H2. Do NOT stuff -- 1-2% density maximum.
- Structure: use H2/H3 hierarchy logically. Every H2 should answer a sub-question of the main query.
- Internal linking: weave 3-6 contextual internal links per 1000 words to related content.
- Schema markup: include Article schema at minimum. Add FAQ or HowTo schema if the content has those patterns.
- Entity-based SEO: reference known entities (people, organizations, concepts) that search engines associate with the topic. Avoid keyword-only approaches.
- Performance awareness: content decisions impact Core Web Vitals. Avoid excessive inline images, unoptimized media, or layout-shifting embeds that would bust LCP/CLS budgets.

**## Discipline Gates** -- 7 gates with full PASS/WARN/FAIL criteria per gate (per D-10, 7 gates not 6):

DISC-SEO-01: Title/H1 Alignment -- Tier 1
- Checks: title tag and H1 convey same primary topic
- Against: asset content structure
- Criteria 1: Title-H1 match (PASS: same topic+keyword, WARN: same topic different phrasing, FAIL: different topics)
- Criteria 2: Title length (PASS: 50-60 chars, WARN: 45-49 or 61-70, FAIL: under 30 or over 70)

DISC-SEO-02: Search Intent Match -- Tier 1
- Checks: content matches the search intent of the target query
- Against: brief's target keyword and intent
- Criteria 1: Intent alignment (PASS: first 100-150 words directly address query, WARN: addresses query but buried below intro, FAIL: content does not match the intent)
- Criteria 2: Content format match (PASS: format matches intent -- listicle for "best X", how-to for "how to X", WARN: partially matches, FAIL: wrong format for intent)

DISC-SEO-03: Schema Markup -- Tier 2
- Checks: structured data presence and correctness
- Against: schema.org specification
- Criteria 1: Schema presence (PASS: Article/FAQ/HowTo schema specified in asset, WARN: schema type mentioned but fields incomplete, FAIL: no schema specified)
- Criteria 2: Schema accuracy (PASS: all required fields present, WARN: optional fields missing, FAIL: required fields missing or incorrect type used)

DISC-SEO-04: Internal Link Density -- Tier 2
- Checks: contextual internal link quantity and quality
- Against: asset content (self-contained check)
- Criteria 1: Link count (PASS: 3-6 contextual internal links per 1000 words, WARN: 1-2 links per 1000 words, FAIL: 0 internal links)
- Criteria 2: Anchor text quality (PASS: descriptive anchor text with context, WARN: generic anchors like "click here" or "read more", FAIL: N/A -- linked to link count)

DISC-SEO-05: Thin Content Detection -- Tier 1
- Checks: content depth and uniqueness
- Against: asset content (self-contained check)
- Criteria 1: Word count (PASS: 800+ words for standard pages, 300+ for pSEO variants, WARN: 500-799 for standard, FAIL: under 500 standard or under 300 pSEO)
- Criteria 2: Unique substance (PASS: content provides original analysis, data, or perspective, WARN: content summarizes existing information with light original insight, FAIL: content is mostly boilerplate or template-generated without substantive content)

DISC-SEO-06: Meta Description -- Tier 2
- Checks: meta description presence, length, and keyword inclusion
- Against: asset metadata
- Criteria 1: Presence and length (PASS: meta description present, 120-160 chars, WARN: present but under 120 or over 160, FAIL: no meta description)
- Criteria 2: Keyword inclusion (PASS: target keyword appears naturally in meta description, WARN: keyword variant or synonym used, FAIL: no keyword or variant present)

DISC-SEO-07: Core Web Vitals Budget -- Tier 1 (per D-10)
- Checks: content structure for CWV impact -- does the asset's content design stay within performance budgets?
- Against: Google Core Web Vitals thresholds (LCP < 2.5s, CLS < 0.1, INP < 200ms)
- Criteria 1: LCP risk (PASS: hero/above-fold content uses optimized image formats (WebP/AVIF) with explicit width/height, lazy-loads below-fold images, no render-blocking embeds above fold, WARN: images present above fold but missing explicit dimensions or using unoptimized formats (PNG/uncompressed JPEG), FAIL: above-fold content depends on heavy unoptimized media (>200KB hero image without dimensions), multiple render-blocking third-party embeds, or auto-playing video without poster frame)
- Criteria 2: CLS risk (PASS: all images and embeds have explicit width/height or aspect-ratio CSS, no dynamically injected content above fold, ad slots have reserved space, WARN: most media has dimensions but 1-2 elements missing explicit sizing, FAIL: images/embeds without dimensions that will cause layout shift, dynamically inserted banners or CTAs above existing content)
- Criteria 3: INP risk (PASS: no heavy inline JavaScript or click handlers that would block interaction -- content relies on standard links and native form elements, WARN: interactive elements present (accordions, tabs) but use standard HTML patterns or lightweight JS, FAIL: content specifies complex interactive widgets, heavy carousels, or inline calculators without performance notes)

**## Base Gate Overrides:**
| Base Gate ID | Default Tier | Override Tier | Reason |
|-------------|-------------|---------------|--------|
| GATE-10 | Tier 2 (advisory) | Tier 1 (blocking) | SEO format correctness directly impacts search rankings; structural errors (missing H1, title) prevent indexing |

**## Format Rules:** Cover SEO-specific structural requirements:
- Title tag: 50-60 characters, target keyword near front
- Meta description: 120-160 characters, compelling with keyword
- H1: exactly one per page, matches title topic
- URL structure: short, hyphenated, includes primary keyword
- Image alt text: descriptive, includes keyword where natural
- Images: explicit width/height attributes, WebP/AVIF format preferred, lazy-load below fold

**## Examples:** Provide 2 good examples and 2 bad examples of SEO content patterns (brief -- 3-4 lines each). Good: intent-matching structure, proper schema. Bad: keyword stuffing, thin pSEO template.

**## Anti-Patterns:** List 6-7 common SEO mistakes:
- Keyword stuffing (density > 2%)
- Title/H1 mismatch (different topics)
- Missing schema markup on eligible content
- Internal links with "click here" anchors
- Duplicate/thin pSEO pages with only location swapped
- Ignoring search intent (commercial query with informational content)
- Hero images without explicit dimensions causing CLS

**## Metrics:** What to measure for SEO content post-ship:
- Organic impressions and clicks (GSC)
- Average position for target keyword
- Click-through rate from SERP
- Time on page / scroll depth
- Internal link click-through
- Core Web Vitals pass rate (PageSpeed Insights / CrUX)

**## Production Guidance** -- How to write AEO-optimized content. Cover:
- SEO cross-reference: open with a callout per CONTEXT.md specific idea: "If this asset also targets organic search, ensure SEO playbook gates are also satisfied."
- Citation-worthiness: write sentences that can be quoted verbatim by AI engines. Use definitive, concise statements. Avoid hedging language ("it might be", "some believe").
- Structure for AI extraction: use clear H2 questions, follow with a 1-2 sentence definitive answer, then elaborate. This maps to how AI engines extract answers.
- FAQ patterns: include explicit question-answer pairs. Use "What is", "How does", "Why" formats that match user queries to AI engines.
- Fact density: concentrate verifiable facts in the first paragraph and section openers. AI engines prioritize high-fact-density passages.
- Multi-source corroboration: ensure claims made can be verified against multiple authoritative sources. AI engines prefer claims that appear consistently across the web.

**## Discipline Gates** -- 5 gates with full PASS/WARN/FAIL criteria:

DISC-AEO-01: Quote-Worthy Sentences -- Tier 1
- Checks: presence of verbatim-quotable standalone answer sentences
- Against: asset content
- Criteria 1: Quote count (PASS: 3+ sentences that could be quoted verbatim as standalone answers by AI engines, WARN: 1-2 quotable sentences, FAIL: 0 quotable sentences -- all sentences require surrounding context)
- Criteria 2: Quote placement (PASS: quotable sentences appear in section openers and H2 answers, WARN: quotable sentences exist but are buried in body paragraphs, FAIL: N/A -- linked to quote count)

DISC-AEO-02: FAQ/HowTo Schema -- Tier 1
- Checks: structured data for AI engine consumption
- Against: schema.org FAQPage/HowTo specification
- Criteria 1: Schema presence (PASS: FAQPage or HowTo schema specified with well-formed Q&A pairs, WARN: schema type specified but incomplete -- missing answers or missing steps, FAIL: no FAQ/HowTo schema on content that has Q&A or step-by-step patterns)
- Criteria 2: Q&A completeness (PASS: every FAQ pair has a substantive answer of 2+ sentences, WARN: some answers are single-sentence or thin, FAIL: FAQ has questions without answers or placeholder answers)

DISC-AEO-03: Author/Expert Markup -- Tier 2
- Checks: author attribution and expertise signals
- Against: asset content and metadata
- Criteria 1: Author schema (PASS: author name and credentials specified in asset metadata or byline, WARN: author name present but no credentials or bio, FAIL: no author attribution)
- Criteria 2: Expertise signals (PASS: content references author's specific experience, data, or research, WARN: generic expertise claim without specifics, FAIL: no expertise signals -- could be written by anyone)

DISC-AEO-04: Cross-Domain Fact Consistency -- Tier 1
- Checks: claims do not contradict other campaign assets
- Against: other assets in the same campaign (MANIFEST.json)
- Criteria 1: Number consistency (PASS: all numeric claims in this asset match the same claims in other campaign assets, WARN: numeric claims use different rounding or timeframes but are directionally consistent, FAIL: numeric claims contradict claims in other campaign assets)
- Criteria 2: Positioning consistency (PASS: key product/service claims are consistent across all campaign assets, WARN: minor phrasing differences that could be interpreted differently, FAIL: direct contradictions in product claims across assets)

DISC-AEO-05: Direct Answer Format -- Tier 2
- Checks: sections open with definitive answers before elaboration
- Against: asset content structure
- Criteria 1: Section opener quality (PASS: each major section (H2) opens with a 1-2 sentence definitive answer to the section's implicit question, WARN: most sections open with answers but some start with context/preamble, FAIL: sections open with context, history, or preamble before answering)
- Criteria 2: Independent citability (PASS: key facts can be extracted from individual paragraphs without needing surrounding context, WARN: some facts require 2+ paragraphs for full context, FAIL: facts are spread across sections with no self-contained extractable statements)

**## Base Gate Overrides:**
None -- all base gates keep default tiers.

**## Format Rules:** Cover AEO-specific structural requirements:
- H2s should be phrased as questions matching user queries ("What is X?" not "About X")
- First paragraph: definitive answer to the page's primary question
- FAQ section: minimum 3 Q&A pairs if using FAQPage schema
- HowTo section: numbered steps with clear action verbs
- Word count: 1,500+ words for comprehensive AEO content (AI engines favor depth)

**## Examples:** Provide 2 good and 2 bad AEO patterns (brief). Good: definitive opener, quotable sentence. Bad: hedging language, buried answer.

**## Anti-Patterns:** List 4-5 common AEO mistakes:
- Opening sections with history/context instead of direct answers
- Hedging language ("It could be argued that...") -- AI engines skip uncertain statements
- FAQ schema with single-word or empty answers
- Copying the same facts in different phrasing (creates self-contradiction risk)
- No author attribution on expertise-dependent content

**## Metrics:** What to measure for AEO content post-ship:
- AI engine citations (manual check across ChatGPT, Perplexity, Gemini)
- Featured snippet captures
- FAQ rich result appearances
- Direct answer box presence in SERPs
