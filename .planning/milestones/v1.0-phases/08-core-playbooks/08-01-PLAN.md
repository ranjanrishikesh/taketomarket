Per D-09, this file defines the shared structure that all discipline playbooks must follow. Per D-03, it defines what sections are required, how gates are structured, and what verify expects to parse.

Content structure:
1. Header explaining this is a contract/template, NOT loaded by produce workflow
2. **YAML Frontmatter Specification** -- document the required frontmatter fields: `discipline` (string), `asset_types` (array), `version` (string "1.0")
3. **Required Sections** -- document all 6 required sections per D-06:
   - `## Production Guidance` -- channel-specific writing instructions for ttm-producer
   - `## Discipline Gates` -- quality checks using DISC-{DISCIPLINE}-{NN} IDs with PASS/WARN/FAIL criteria
   - `## Base Gate Overrides` -- table mapping base gate IDs to new tiers (per D-02). Format: `| Base Gate ID | Default Tier | Override Tier | Reason |`. If no overrides, include the section with "None -- all base gates keep default tiers."
   - `## Format Rules` -- platform-specific constraints
   - `## Examples` -- good/bad patterns (extract to references/playbook-{discipline}-examples.md if >500 lines per D-07)
   - `## Anti-Patterns` -- common mistakes
   - `## Metrics` -- what to measure post-ship for this channel
4. **Gate Definition Format** -- document the exact structure each discipline gate must follow, matching base-gates.md pattern but using DISC-{DISCIPLINE}-{NN} IDs per Pattern 2:
   ```
   ### DISC-{DISCIPLINE}-{NN}: {Name} -- Tier {1|2}
   **Checks:** {what is evaluated}
   **Against:** {reference data}
   #### Evaluation Criteria
   1. **{Check name}**
      - PASS: {condition}
      - WARN: {condition}
      - FAIL: {condition}
   ```
5. **Tier Rules** -- Tier 1 discipline gates trigger Correct/Accept+log/Escalate deviation handling. Tier 2 are advisory.
6. **Override Rules** -- per D-01, base gates always run; discipline gates run additionally. Per D-02, overrides adjust tier only -- they cannot disable a base gate.
7. **500-Line Limit** -- per D-07, discipline playbooks must stay under 500 lines. If exceeding, extract Examples or Anti-Patterns to reference files.

Keep base.md under 150 lines. It is a contract, not content.
